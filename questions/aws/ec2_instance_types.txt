Para que servem as instâncias General Purpose na AWS?
São instâncias equilibradas em CPU, memória e rede, ideais para uma variedade de casos de uso gerais.

Quando devo usar Compute Optimized Instances na AWS?
Quando você precisa de alto desempenho de CPU, como em processamento em lote, transcodificação de mídia e jogos dedicados.

Qual o foco das instâncias Memory Optimized da AWS?
São ideais para workloads com uso intensivo de memória, como análise em tempo real de Big Data, cache in-memory e bancos de dados open source.

O que são Accelerated Computing Instances na AWS?
Instâncias com aceleradores de hardware (como GPU), voltadas para machine learning, reconhecimento de voz, gráficos e veículos autônomos.

Quando usar Storage Optimized Instances na AWS?
Quando o workload exige altas taxas de leitura e escrita em grandes volumes de dados, como bancos de dados transacionais, motores de busca e streaming de dados.

Para que servem as instâncias HPC (High Performance Computing)?
São otimizadas para cargas de trabalho de HPC em larga escala, como previsão do tempo, simulações moleculares e testes de colisão.
